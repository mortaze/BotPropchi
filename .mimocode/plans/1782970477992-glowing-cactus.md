# Plan: Admin Rate Limit Bypass + Persistent State

## Problem
1. Admins hit rate limits (20 req/60s) like regular users — no bypass exists
2. Admin editing state (menu editor, post editor) expires after 300s/600s cache TTL, forcing `/start` restart

## Change 1: Rate Limit Bypass for Admins

### File: `src/bot/middlewares/index.ts`

**Add an in-memory admin ID set** at module level (next to `userRequestCounts`):

```typescript
const adminIds = new Set<number>();
```

**Add a refresh function** that loads all ACTIVE BotAdmin telegramIds from DB:

```typescript
import { prisma } from '../../prisma/client';
import { BotAdminStatus } from '@prisma/client';

async function refreshAdminIds() {
  try {
    const admins = await prisma.botAdmin.findMany({
      where: { status: BotAdminStatus.ACTIVE },
      select: { telegramId: true },
    });
    adminIds.clear();
    for (const a of admins) adminIds.add(Number(a.telegramId));
  } catch {}
}
```

**Call `refreshAdminIds()` once at module init** and **setInterval every 5 minutes**:

```typescript
refreshAdminIds();
setInterval(refreshAdminIds, 5 * 60 * 1000);
```

**Modify `rateLimitMiddleware`** — add admin bypass at the top of the function, before the counting logic:

```typescript
export function rateLimitMiddleware(maxRequests = 20, windowMs = 60_000) {
  return async (ctx: Context, next: () => Promise<void>) => {
    if (!ctx.from) return next();
    const userId = ctx.from.id;
    
    // Admins bypass rate limiting entirely
    if (adminIds.has(userId)) return next();
    
    // ... existing counting logic unchanged ...
  };
}
```

### Why this approach
- Rate limiter runs at position 2 in middleware chain (before userMiddleware), so we can't call `botAdminService.getActive()` (DB query per message)
- The in-memory Set is populated once and refreshed every 5 min — negligible overhead
- Covers ALL admin roles (OWNER, SUPER_ADMIN, ADMIN, MODERATOR)
- Set is automatically cleared on bot restart (safe — no stale data)

## Change 2: Persistent Admin State (No Cache Expiry)

### File: `src/utils/cache.ts`

**Add `setPermanent` method** to `CacheService`:

```typescript
setPermanent<T>(key: string, value: T): void {
  this.cache.set(key, value, 0); // TTL=0 = no expiry in node-cache
}
```
