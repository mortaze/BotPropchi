# UI/UX Redesign Plan — Login + Responsive Analytics

## Scope

Frontend-only changes. No backend, API, route, validation, logic, field names, state, event, or request changes.

| # | File | Change |
|---|------|--------|
| 1 | `admin/src/app/Globals.css` | Add `.input`, `.label` base styles for login form consistency |
| 2 | `admin/src/app/login/page.tsx` | Full visual redesign — modern SaaS login |

## Design System Decisions

- **Style**: Minimal dark SaaS (Linear/Vercel/Stripe Dashboard level)
- **Colors**: Use existing CSS variables (`--primary`, `--card`, `--border`, etc.) — no new colors
- **Typography**: Vazirmatn (already configured). Hierarchy: 2xl bold title → sm muted subtitle → xs label → sm input → sm button
- **Shadows**: Subtle (`shadow-lg shadow-primary/5` on card, `shadow-sm` on button active)
- **Border radius**: `rounded-2xl` card, `rounded-xl` inputs, `rounded-lg` button (match existing `--radius: 0.625rem`)
- **Animations**: `transition-all duration-200` for interactions, `animate-fade-in` for page load

## Task 1: Login Page Redesign

### Current Issues
- Card uses `rounded-2xl border border-border bg-card p-8 shadow-2xl` — shadow too heavy
- Inputs use raw `<input>` with manual icon positioning — not using `Input` component from `@/components/ui`
- No consistent `.input` / `.label` CSS classes defined in Globals.css (login uses them but they're undefined)
- Button uses `min-h-10` from Button component — good, but card layout needs polish
- Password toggle button alignment could be tighter

### New Design
1. **Full-screen gradient background**: Subtle radial gradient using `--sidebar` (dark blue) → `--background` blend, with a faint grid/dot pattern via CSS
2. **Login card**: `max-w-sm` (narrower), centered, `rounded-2xl`, subtle `shadow-2xl shadow-black/20`, `border border-white/[0.08]` for glass effect
3. **Logo area**: Larger icon container with gradient background, brand name in `text-2xl font-bold tracking-tight`, subtitle in `text-sm text-muted-foreground`
4. **Inputs**: Use the project's `Input` component from `@/components/ui` — standardized height, border-radius, focus ring, and error states. For password field, wrap in a relative container with the eye toggle positioned inside.
5. **Submit button**: Full-width, `h-11`, `rounded-xl`, gradient primary background, `shadow-lg shadow-primary/25`, hover lift effect
6. **Footer text**: Small muted text below card for branding
7. **Responsive**: `p-4` on mobile, card scales cleanly

### Specific CSS Changes (Globals.css)
- Add `.input` class: `w-full h-11 px-4 rounded-xl border border-input bg-background text-sm transition-all duration-200 focus:outline-none focus:ring-2 focus:ring-primary/20 focus:border-primary placeholder:text-muted-foreground/60`
- Add `.label` class: `block text-sm font-medium text-foreground mb-1.5`

### Login Page JSX Rewrite
- Replace manual `<input>` elements with the `Input` component from `@/components/ui`
- Wrap password field in a `relative` div with the eye icon button positioned `left-3`
- Card wrapper: `max-w-sm w-full mx-auto rounded-2xl border border-white/[0.08] bg-card/80 backdrop-blur-xl p-8 shadow-2xl shadow-black/20`
- Background: `min-h-screen flex items-center justify-center bg-background relative overflow-hidden` with a pseudo-element for gradient

## Task 2: Acquisition Page — Responsive Time Range

### Current Issue
Lines 185-198: The time range section is a single flex row with preset buttons + two JalaliDatePicker components. On mobile (< 640px), this overflows horizontally.

### Solution: Mobile Bottom Sheet Pattern
- **Desktop (sm+)**: Keep current layout (presets + date pickers in a row)
- **Mobile (< sm)**: Show a single "بازه زمانی" button. On click, open a bottom sheet/modal with:
  - Preset buttons (stacked vertically)
  - "بازه دلخواه" option that reveals start/end date pickers
  - Selected range displayed below the button as text

### Implementation
1. Create a `TimeRangeSelector` component (inline in the file, not a separate file) that handles both layouts
2. Use the existing `Modal` component from `@/components/ui` for the mobile bottom sheet (adapt with `size="sm"` and position from bottom)
3. After selection, show: `بازه: {startJalali} تا {endJalali}` or preset label below the button

### Mobile Layout
```
[ بازه زمانی ▼ ]          ← single button
بازه: ۱۴۰۴/۰۴/۰۱ تا ۱۴۰۴/۰۴/۱۵  ← selected range text
```

### Desktop Layout (unchanged)
```
[ ۷ روز ] [ ۲۸ روز ] [ ۳ ماه ]  از: [date] تا: [date]
```
