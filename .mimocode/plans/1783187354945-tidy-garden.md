# Plan: Complete Architectural Independence — Auto-Reply Module

## Current Coupling

27 shared `bot.hears()` text patterns + identical keyboard labels between auto-reply and scheduled-message. Both modules produce the same Persian text for internal actions (create, list, report, publish, delete, etc.). Telegraf first-match dispatch makes auto-reply unreachable.

## Goal

Zero runtime dependency. Auto-reply becomes a fully independent clone with its own labels, entry point, and keyboard output. Scheduled-message: ZERO changes.

### 2. `src/bot/handlers/auto-reply.handlers.ts` — Match new labels + independent entry

Update every `bot.hears()` to match the new keyboard labels. Update `knownButtons` array.

Add `/autoreply` command as the sole entry point (replacing the shared automation menu entry):

```typescript
bot.command('autoreply', async (ctx: any) => {
  const admin = await botAdminService.getActive(ctx.from.id);
  if (!admin) return;
  autoReplyState.clearAll(ctx.from.id);
  autoReplyState.setManagementMode(ctx.from.id, true);
  const result = await autoReplyRepository.findAll({ page: 1, limit: 100 });
  await ctx.reply('💬 پاسخ‌های خودکار', autoReplyMainMenuKeyboard(result.items));
});
```

Remove the two dead entry handlers:
- `bot.hears('🤖 اتوماسیون', ...)` (line 66) — remove entirely
