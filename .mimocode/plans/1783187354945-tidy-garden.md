# Plan: Auto Reply → Keyword-Based System

## Overview

Transform Auto Reply from time-scheduled to keyword-based matching. When a user sends a message in a group containing a registered keyword, the bot replies to that message with the Auto Reply's post content.

## 1. Database Schema

**New model: `AutoReplyKeyword`** in `prisma/schema.prisma`:

```prisma
model AutoReplyKeyword {
  id          Int       @id @default(autoincrement())
  autoReplyId Int
  keyword     String
  createdAt   DateTime  @default(now())
  autoReply   AutoReply @relation(fields: [autoReplyId], references: [id], onDelete: Cascade)

  @@index([autoReplyId])
  @@map("auto_reply_keywords")
}
```

Add relation to `AutoReply`:
```prisma
model AutoReply {
  // ... existing fields ...
  keywords    AutoReplyKeyword[]  // ADD THIS
}
```

**Remove scheduling fields from `AutoReply`** (keep `targetChatId`/`targetTopicId` for reply destination):
- Remove: `intervalMinutes`, `startTime`, `lastSentAt`, `nextSendAt`, `slug`
- Keep: `targetChatId`, `targetTopicId` (for knowing where to send in groups)

Run `npx prisma db push` after schema change.

## 3. Keyword Matching Logic

In `auto-reply.service.ts`, new method `handleGroupMessage(ctx, telegramGroupId)`:

```
1. Extract text from message (text or caption)
2. If no text, return false
3. Query all published AutoReplies with keywords (include keywords)
4. For each AutoReply:
   a. For each keyword:
      - Case-insensitive check: text.toLowerCase().includes(keyword.toLowerCase())
   b. If match found:
      - Build inline keyboard from AutoReply's buttons
      - Reply to user's message with AutoReply's messages (same logic as current sendToGroup)
      - Log the delivery
      - Return true
5. If no match, return false
```
